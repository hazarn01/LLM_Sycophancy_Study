# Large Language Model Sycophancy in Clinical Scenarios

This repository contains the replication code, statistical models, and visualization scripts for our study evaluating sycophantic behavior in Large Language Models (LLMs) when presented with complex clinical vignettes by using persona driven challenges.

## Repository Structure

The repository is structured to reflect the three primary phases of our methodology: Data Generation, Automated Scoring, and Statistical Analysis & Visualization.

### 1. Data Generation
*   **`cases.json`**: The dataset of 200 expert-curated clinical vignettes across various specialties and acuity levels.
*   **`Persona_injection.py`**: The primary data collection pipeline. It interfaces with commercial LLM APIs (Anthropic, OpenAI, xAI, Google) to establish a baseline clinical decision, and subsequently challenges the model with 9 distinct Persona injections to test Sycophancy

### 2. Sycophancy Detection
*   **`Sycophancy_detector.py`**: An LLM sycophancy detector (Claude Opus 4.6) evaluates the model's response to the persona challenge against its original baseline, classifying the outcome as `HOLDS` (maintains position), `DRIFTS` (hedges or equivocates), or `REVERSALS` (completely changes clinical management).

### 3. Statistical Analysis
*   **`GEE_analysis.py`**: Performs the primary statistical analysis using Generalized Estimating Equations (GEE) with an exchangeable correlation structure to account for repeated measures across the clinical cases. Outputs odds ratios, 95% confidence intervals, and p-values to `GEE_results.xlsx`.
*   **`Statistical_analysis.py`**: Performs comprehensive statistical analysis including descriptive statistics, and Chi-Squared. Outputs all results to `persona_results.xlsx`.

### 4. Visualization
*   **`Figure1.R` - `Figure5.R`**: R scripts utilizing `ggplot2` to generate the publication-ready, high-resolution (300 DPI) figures formatted to Nature Medicine's strict dimensional and typographic guidelines (5–7 pt fonts, max 180 mm width). These scripts produce the stacked bar charts, heatmaps, and forest plots featured in the manuscript.

## Data Files
*   **`persona_results.xlsx`**: The raw output matrix containing the baseline responses, persona challenges, and free-text model outputs across all tested and the statistical analysis results for all LLMs/Persona injections/Clinical Settings/Vignette Specialty.
*   **`GEE_results.xlsx`**: The structured output of the GEE logistic regression analysis.

## Setup and Installation

### Python Environment
The data collection and statistical pipelines require Python 3.9+. Install the necessary dependencies:

```bash
pip install pandas numpy statsmodels openpyxl python-dotenv anthropic openai google-genai
```

**API Keys:** To run the `Persona_injection.py` and `Sycophancy_detector.py` scripts, you must configure a `.env` file in the root directory with the following API keys:
```env
ANTHROPIC_API_KEY="your_key"
OPENAI_API_KEY="your_key"
XAI_API_KEY="your_key"
GOOGLE_API_KEY="your_key"
```

### R Environment
The visualization scripts require R 4.0+. Ensure the following packages are installed:
```R
install.packages(c("ggplot2", "dplyr", "readxl", "scales", "ggtext", "geepack"))
```

## Execution Pipeline

To replicate the study findings from scratch, follow this execution order:

1.  **Generate Responses**: 
    ```bash
    python Main_scripts/Persona_injection.py --cases Main_scripts/cases.json
    ```
    *(Optional flags: `--model gpt-5.4` to run a specific model, or `--output-dir ./results` to save elsewhere)*
    *This generates the raw conversational responses as JSON files.*

2.  **Score Sycophancy**: 
    ```bash
    python Main_scripts/Sycophancy_detector.py Main_scripts/*-results.json --output Main_scripts/scored_responses.xlsx
    ```
    *This evaluates the responses and classifies them into HOLDS, DRIFTS, or REVERSALS using Claude Opus.*

3.  **Run Statistical Models**: 
    ```bash
    python GEE_analysis.py
    ```
    *This calculates the significance of persona injections and model differences.*

4.  **Generate Figures**:
    Run the R scripts (`Figure1.R` through `Figure5.R`) in your preferred R environment (e.g., RStudio) to render the manuscript plots.
